# 01. 개요

## 1.1 문제 정의

사내에서 **Claude Code 를 Amazon Bedrock 백엔드로** 사용하면서, 토큰 사용량에 비례한 비용이
누적되고 있다. 특히 다음 유형의 작업이 토큰을 크게 소모한다.

- **대량 코드 생성** — 보일러플레이트, CRUD, 구현체 다수, 스캐폴딩
- **테스트 작성** — 반복적·패턴적이며 출력량이 많음
- **일괄 리팩터링/이식** — 여러 파일에 동일 변환
- **대용량 파일/로그 분석** — 수천 줄을 컨텍스트로 읽어들임

이런 작업은 **고난도 추론이 필요하지 않은데도** 비싼 Bedrock 출력/입력 토큰을 많이 쓴다.

## 1.2 핵심 통찰

Bedrock(Claude) 과금 구조에서 **출력 토큰 단가가 입력보다 약 5배 높다**
(예: Sonnet $3/1M 입력 vs $15/1M 출력). 따라서:

- 길게 **생성**되는 작업 → 출력 토큰 비용이 지배적
- 큰 파일을 **읽는** 작업 → 입력 토큰 비용이 지배적

두 비용 모두, 작업 자체가 "기계적"이라면 **로컬 코드 특화 LLM(Ollama)** 으로 옮길 수 있다.
로컬 모델은 무료(전기/하드웨어 비용만)이고, 코드 생성 같은 정형 작업에서 충분한 품질을 낸다.

## 1.3 목표

1. **비용 절감(3기둥 + 역할 분담)** — ① 코드 탐색은 지식 그래프(codebase-memory-mcp)로(입력↓),
   ② 개발 대부분은 실행자 executor(사내 GLM-5.2, 폴백 H200/V100)로(출력↓, 무제한),
   ③ 고난도 판단·검토만 조언자 advisor(Claude/Bedrock, $200/월 예산). 비용 사다리에서 충분한
   가장 싼 단계를 쓴다. → [13](13-multi-model-agents.md) · [18](18-executor-advisor.md)
2. **보안 우선 + 품질 보존** — **기밀이 감지되면 판단까지 사내 GLM-5.2 가 담당**(Bedrock 전송
   금지). 기밀 없는 설계/복잡 디버깅 등 고난도 판단은 Claude 가 수행하고, 위임 결과도 Claude 가
   최종 검토.
3. **마찰 최소화** — Claude Code 스킬/서브에이전트/훅으로 위임을 **자동·반자동**화.
4. **측정 가능성** — 위임 건수·토큰·환산 절감액을 로깅해 효과를 추적.

## 1.4 비목표 (Non-goals)

- Claude 를 완전히 대체하는 것이 아니다. **역할 분담**이 목적이다.
- 로컬 모델로 아키텍처 설계·보안 판단을 대신하지 않는다(품질·위험).
- Bedrock 사용을 강제로 차단하지 않는다(가용성 우선, 실패 시 Claude fallback).

## 1.5 절감 원리 세 가지

### (A) 탐색 위임 — 입력 토큰 절감 [기본·codebase-memory-mcp]

```
[기존]  Claude 가 코드 이해하려 파일 수십 개를 read/grep  →  비싼 입력 토큰 폭증
[TokenLift]  지식 그래프에 구조 쿼리(get_architecture/search_graph/trace_path)
            ⇒ 5개 쿼리 ≈ 3,400 토큰 vs 파일 탐색 ≈ 412,000 토큰 (약 99%↓)
```

### (B) 생성 위임 — 출력 토큰 절감 [Ollama/NemoClaw]

```
[기존]  Claude 가 200줄 코드 직접 생성  →  비싼 출력 토큰 200줄분 과금
[TokenLift]  로컬/온프렘 모델이 200줄 생성  →  Claude 는 결과만 검토(짧은 입력)
            ⇒ 비싼 출력 토큰을 거의 0 으로
```

### (C) 비코드 컨텍스트 위임 — 입력 토큰 절감 [Ollama explain]

```
[기존]  Claude 가 5,000줄 로그 전체를 읽음  →  비싼 입력 토큰 5,000줄분
[TokenLift]  사내 Ollama 서버가 읽고 10줄 요약  →  Claude 는 요약만 받음
            ⇒ 입력 토큰을 1/500 수준으로
```

> (A)는 **코드** 탐색(구조적), (C)는 **비코드** 대용량 텍스트(로그 등) 요약에 쓴다.

## 1.6 전체 그림

```
                        ┌──────────────────────────────────┐
   사용자 요청  ───────► │   Claude Code (Bedrock)            │
                        │   - 의도 파악/오케스트레이션 + 기밀 검증 │
                        │   - (비민감) 설계/복잡 디버깅 = 조언자   │
                        │     기밀이면 판단까지 사내 GLM-5.2 로   │
                        └─────┬───────────────┬──────────────┘
            코드 탐색이 필요    │               │  무거운 생성 작업만 잘라서
                        ▼               ▼
        ┌───────────────────────┐   ┌─────────────────────────┐
        │ codebase-memory-mcp   │   │   tokenlift CLI (브리지)  │
        │  지식 그래프(로컬)      │   │   - 태스크→provider/모델   │
        │  get_architecture     │   │   - 프롬프트/결과 정제      │
        │  search_graph/trace   │   │   - 절감 로깅              │
        │  get_code_snippet     │   └───────────┬─────────────┘
        └──────────┬────────────┘               ▼
                   │ 구조 답변          ┌─────────────────────────┐
                   │ (입력 토큰↓)        │  Ollama(로컬) / NemoClaw  │
                   │                   │  (NIM, 온프렘) — 출력↓     │
                   │                   └───────────┬─────────────┘
                   └────────────┬──────────────────┘ 결과(코드)
                                ▼
                     Claude 가 검토·통합·책임
```
