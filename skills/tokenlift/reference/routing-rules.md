# 위임 판단 상세 규칙

이 문서는 "무엇을 사내 모델로 보내고 무엇을 Claude가 처리할지"의 의사결정 기준이다.
핵심 목표: **기밀은 사내를 벗어나지 않게, 비싼 출력/입력 토큰은 로컬로 옮기되,
정확성과 판단이 중요한 부분은 Claude(조언자)가 책임진다.**

## 의사결정 순서 (기밀 우선)

0. **기밀(민감) 신호가 있는가?** → 있으면 **사내 GLM-5.2 강제**(executor/oracle),
   Bedrock 프롬프트에 기밀 원문 금지. `tokenlift route` 가 기밀도를 판정한다.
1. **고난도 판단 신호가 있는가?**(설계/근본원인/의사결정) → 비민감이면 Claude(advisor).
2. **위임 적합 신호 + 임계값 초과인가?** → executor(GLM-5.2)/coder(V100)로 위임.
3. **애매하면** → `tokenlift route` 추천 참고, 비민감 기본은 Claude.

## Ollama로 위임 (✅ 적극 위임)

| 패턴 | 예시 요청 | 권장 명령 |
|---|---|---|
| 명세 명확한 신규 코드 | "이 인터페이스 구현체 작성" | `gen` |
| 보일러플레이트/스캐폴딩 | "CRUD 컨트롤러 5개 골격" | `gen` |
| 단위 테스트 | "이 모듈 테스트 작성" | `test` |
| 일괄 리팩터링 | "콜백을 async/await로", "함수 분리" | `refactor` |
| 언어/프레임워크 이식 | "이 파이썬을 Go로" | `translate` |
| 대용량 파일/로그 요약 | "이 5천 줄 로그에서 에러 원인" | `explain` |
| docstring/주석/문서 초안 | "공개 함수에 JSDoc" | `docs` |
| 정형적 변환 | "JSON 스키마 → 타입 정의" | `gen`/`translate` |

**위임 임계값(휴리스틱):** 생성 코드 30줄 이상, 처리 파일 300줄 이상, 동일 패턴 3개 파일 이상.
이 미만의 사소한 작업은 위임 왕복 비용(지연)이 절감액보다 클 수 있어 Claude가 바로 처리해도 된다.

## 사내 강제 (🔒 기밀 — Bedrock 전송 금지)

| 패턴 | 처리 |
|---|---|
| 개인키/시크릿/토큰이 포함된 코드·설정 | executor(GLM-5.2) — 외부 전송 금지 |
| `.internal` 호스트·사내 도메인 정보 | 〃 |
| 개인정보(주민번호 등)·고객 데이터 | 〃 |
| "사내기밀/대외비" 문서·프로젝트 코드명(`security.sensitivePatterns`) | 〃 |
| 위 내용에 대한 설계/보안 "판단"까지 필요 | **oracle(GLM-5.2)** — Claude 에겐 기밀 제거한 추상 질문만 |

## Claude(advisor)가 처리 (비민감일 때만)

| 패턴 | 예시 요청 | 이유 |
|---|---|---|
| 시스템/서비스 아키텍처 설계 | "전체 결제 시스템 설계" | 고난도 판단, 광범위 영향 |
| API/인터페이스/스키마 설계 | "공개 API 계약 정의" | 하위 호환·일관성 |
| 복잡한 디버깅·근본원인 분석 | "왜 간헐적으로 실패하나" | 다중 컴포넌트 추론 |
| 보안 로직의 "판단"(기밀 원문 제거 후) | 인증/인가/암호화/결제 검토 | 정확성·위험 |
| 전체 시스템 영향 리팩터링 계획 | "도메인 모델 전면 개편" | 교차 의존성 |
| 요구사항 모호·트레이드오프 결정 | "어떤 방식이 나을까" | 맥락·책임 |
| **위임 결과의 최종 검토/통합** | (항상) | 품질 게이트 |

> Claude(Bedrock)는 **$200/월 예산 자원** — 판단·조언·검토만. 긴 코드 생성은 항상 executor 로.

## 하이브리드 패턴 (권장)

대부분의 실제 작업은 "조언(설계)은 Claude, 실행은 executor(GLM-5.2), 검토는 Claude"로 나뉜다.

```
사용자: "알림 모듈 만들어줘 (이메일/SMS/푸시 채널)"

0. route   : 기밀 검증(기밀이면 1·4 도 사내 GLM 이 담당)
1. Claude  : 인터페이스·추상화·에러 전략 설계 (고난도, 짧은 출력)
2. executor: 각 채널 구현체 3종 생성   → tokenlift gen ... --role executor
3. executor: 각 구현체 테스트 생성     → tokenlift test ... --role executor
4. Claude  : 생성물 검토·보안 점검·통합 (짧은 판단)
```

이렇게 하면 가장 비싼 부분(2,3의 대량 코드 출력)이 사내에서 처리된다.

## 모델 선택 요약 (task → 역할 → model)

| task | 역할(기본) | 기본 모델 | 비고 |
|---|---|---|---|
| gen/edit/test/refactor/translate/review | **executor** | `glm-5.2-nvfp4`(Blackwell)/`glm-5.2-fp8`(H200) | 개발 주력·기밀 안전 |
| reason / agent | oracle | 〃 (`--think on`) | 어려운 추론 |
| explain | coder | `qwen3:8b`(V100) | 경량 요약 |
| docs | coder | `llama3.1:8b`(V100) | 산문 |
| fast (간단/저지연) | coder | `gemma3:12b`(V100) | 경량 |
| complete (FIM) | coder | `qwen2.5-coder:1.5b-base`(V100) | 중간채우기 |

`-m`/`--role`/`--provider` 로 강제 지정 가능. 모델을 자주 바꾸면 재적재 비용이 크므로,
한 세션에선 같은 역할(모델)로 묶어 위임하는 것이 빠르다.

## 안티패턴 (하지 말 것)

- ❌ **기밀(키/고객정보/코드명) 원문을 Bedrock 프롬프트에 포함** — route 가 금지로 판정
- ❌ 보안/결제 코드를 위임하고 검토 없이 그대로 사용
- ❌ 5줄짜리 사소한 수정을 위임(왕복 지연 > 절감)
- ❌ 모호한 요구를 그대로 위임(로컬 모델이 엉뚱하게 해석)
- ❌ 매 호출마다 다른 모델 사용(재적재로 느려짐)
- ❌ 위임 산출물을 Claude가 다시 전량 재생성(절감 효과 상쇄 + 예산 낭비)
